Exciting Coding

이해가 안되는 부분은 마법이라 생각해도 괜찮다.

팀 버너스리는 두 대의 컴퓨터를 장만한다. 또 두 개의 프로그램을 개발한다.

하나는 web browser, 다른 하나는 web server.

한 컴퓨터엔 web browser, 다른 컴퓨터엔 web server가 깔려있다.

두 컴퓨터는 인터넷으로 연결되어 있으며, server가 깔려있는 컴퓨터는 http://info.cern.ch라는 주소를 갖고있다.

web server가 깔린 컴퓨터의 하드 디스크에 index.html 문서가 저장되어있다.

web browser가 설치된 컴퓨터가 http://info.cern.ch/index.html을 주소창에 입력하고 enter를 땅 치면, 인터넷을 통해 전기적인 신호를 web server가 설치된 컴퓨터에 보낸다.

그 신호 안에는 '난 index.html을 원한다.'라는 의미가 담겨있다.

그럼 web server가 설치된 컴퓨터는 index.html을 찾아서 전기적 신호로 바꾼 후 보내준다.

web browser가 설치된 컴퓨터엔 index.html의 내용이 담긴 코드가 도착하고 그것을 해석하고 화면에 표시하면 우리가 흔히 알고 있는 웹사이트가 동작하는 것이다.

두 컴퓨터는 서로 정보를 주고 받고 있다.

web browser가 설치된 컴퓨터는 정보를 요청(request)하고 있고, web server가 설치된 컴퓨터는 정보를 응답(response)하고 있다.

요청하는 컴퓨터를 클라이언트client컴퓨터라 부르고, 응답하는 컴퓨터를 서버server컴퓨터라 부른다.
